# Security and Public/Private Boundary

## Purpose

This public repository is a sanitized mirror intended for portfolio and technical evaluation.

## Public Content

The following content is safe to publish:
- architecture and operating model documentation
- abstracted workflow descriptions
- synthetic or sanitized example artifacts
- non-sensitive schemas and templates

## Private Content (Not Published Here)

The private full repository retains:
- internal implementation prompts and agent internals
- environment-specific connectors and operational scripts
- sensitive project details and proprietary logic
- full historical session records

## Access Strategy for Employer Technical Review

1. Share this public mirror as the default artifact.
2. For serious opportunities, invite selected reviewers to the private repository with read-only permissions.
3. Limit access to named users and remove after process completion.

## Governance Notes

- Never publish credentials, tokens, hostnames, or customer-identifying details.
- Maintain a clear boundary between public design documentation and private operational implementation.
- Keep examples synthetic unless explicitly approved for disclosure.
